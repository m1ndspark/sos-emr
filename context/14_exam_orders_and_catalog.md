# Exam Orders + Exam_Catalog Reference Table

First established 2026-07-14. Companion to the X_Ray_Orders form. The live app is
source of truth; this records the agreed design and the Exam_Catalog build.

--------------------------------------------------------------------------------
1. DECISION: X_Ray_Orders repurposed as a self-authored imaging order
--------------------------------------------------------------------------------
X_Ray_Orders and Encounter_LabRequest were originally built as routing
destinations for X-ray-only or lab-only referrals. That model is dropped.

New model: X_Ray_Orders is a SELF-AUTHORED diagnostic imaging order that SOS
staff complete (X-ray, ultrasound, and cardiac studies, matching the field
vendors we use). On submit it:
  1. mints the order ID,
  2. generates a hard-copy order PDF,
  3. attaches the PDF to the order record and to the linked referral,
  4. auto-transmits the order to the selected imaging vendor.

Patient, referral, and partner data pull and lock from Referral_Link (the PVS
pull pattern). Referral_ID and partner do NOT live on the reference tables; they
live on the order record via Referral_Link.

--------------------------------------------------------------------------------
2. TRANSMIT DESIGN: fax-first
--------------------------------------------------------------------------------
Imaging vendors have no API. They receive orders two ways: a website portal and
fax. Only one vendor accepts our own PDF; the rest require their own intake form.
Fax is the universal channel, so:
  - Assume every vendor accepts fax.
  - Submit ALWAYS generates the hard-copy PDF regardless of vendor.
  - One SRFax path faxes that PDF to the vendor fax on file. This single path
    covers every vendor.
  - Portal auto-entry (filling a vendor's web form) is a later OPTIONAL
    browser-agent job, not core.

Planned vendor directory form (new) carries per vendor: name, fax, email,
accepted format (our PDF vs portal), portal URL, active. A lookup on the order
selects the vendor so the fax destination travels with the selection.

--------------------------------------------------------------------------------
3. EXAM_CATALOG REFERENCE FORM - BUILT + VERIFIED LIVE (59 rows)
--------------------------------------------------------------------------------
Reference table for the order's procedure picker. Mirrors the ICD10_Codes
pattern: a reference form + a multi-select lookup + a print rollup. No workflows
on the form.

Form link name: Exam_Catalog. Fields (link names match the import CSV headers):
  - Exam_Code           Single Line, No-duplicate ON  (the unique key)
  - CPT_Code            Single Line (TEXT, not Number; holds dual codes such as
                        76641/76642 and must not drop anything)
  - Exam_Description    Single Line
  - Exam_Display        Single Line ("71046 - Chest AP/LAT 2 View", the field the
                        lookup searches and shows)
  - Modality            Radio: X-Ray, Ultrasound, Cardiac
  - Body_Region         Single Line (text, to avoid choice-match failures on
                        import, same call as ICD Chapter/Category)
  - Laterality_Applicable  Radio: Yes, No
  - Status              Radio: Active, Retired (initial value Active)

DATA: 59 exams (37 X-Ray, 18 Ultrasound, 4 Cardiac), curated from the XR Mobile
provider order form study list. This is a curated "what we order" pick-list, NOT
a full CPT set: no CT, MRI, nuclear medicine, PET, mammography, fluoroscopy, or
bone density. Import CSV generated by Claude: Exam_Catalog_Import.csv (kept as the
seed artifact, not committed, same handling as ICD10_Codes_Import_FY2027.csv).

DESIGN NOTES:
  - CPT_Code is NOT unique (Renal and Bladder are both 76770; Venous Upper and
    Venous Lower are both 93970/93971), which is why Exam_Code is the unique key,
    not CPT.
  - One row per exam plus a Laterality_Applicable flag (25 exams flagged Yes).
    Laterality is NOT exploded into separate L/R/Bilat rows. If the picker later
    needs per-side entries, regenerate exploded rows from the source in one pass.
  - Exams with no CPT on the source form (Neck Soft Tissue, Complete Abdominal,
    Soft Tissue Groin, Rule out DVT) have blank CPT_Code and display by name only.

Order field superset (confirmed against the XR Mobile and RapidRad vendor forms):
  ordering provider + NPI + phone + fax + email + order date + typed-name
  signature with medical-necessity attestation + priority (Routine/STAT); patient
  name, DOB, gender, SSN last 4, room, phone, facility, address (pull-lock from
  Referral_Link); insurance primary and secondary (the referral MBI is usually
  the Medicare primary ID); reason for study + special instructions.

first; staff alerting deferred. Acknowledged/Completed dropped (not knowable from a fax).
